## `DashBoard.tsx` 페이지
- 상태 변화 시 기기의 `device.status`를 알려줍니다.
  - 한계: 어떤 기기의 상태가 변했는지 알 수 없고, 내용에 `{device.name}`을 명시해줘야합니다. 그리고 `aria-atomic="true"`를 추가해서 전체 상태를 읽도록 보장해야합니다.
```
<Status aria-live="polite">
    상태: {device.status}
</Status>
```
- 기기타입이 "light"인 상태의 버튼 기능 및 상태를 명시합니다.
```
<ToggleButton
    ...
    aria-label={device.status === 'online' ? `${device.name} 전원 끄기` : `${device.name} 전원 켜기`}
    aria-pressed={device.status === 'online'}
>
```

## `DetailDevice.tsx` 페이지
#### 1. `PowerButton` 컴포넌트
- 버튼이 하는 일을 "전원 켜기", "전원 끄기"라고 구체적으로 설명합니다.
- 버튼을 클릭하기 전에 그 기능을 정확히 인지하게 하고, 클릭시 전원 상태를 스크린 리더에게 전달합니다.
```
<IconButton
    ...
    aria-label={power === "on" ? "전원 끄기" : "전원 켜기"}
    aria-pressed={power === "on"}
/>
```

#### 2. `Temperature` 컴포넌트
- 온도 조절 버튼이 하는 일을 "1도 하락", "1도 상승"이라고 구체적으로 설명합니다.
- 상태 변환 후 변한 `{temperature}`값을 알려줍니다. `aria-atomic`속성으로 영역전체를 한 번에 읽습니다.
```
<CircleButton 
    ...
    aria-label="1도 하락"
>
    <TempText
        aria-live="polite" 
        aria-atomic="true"
    >
        {temperature}
        <TempUnit>℃</TempUnit>
    </TempText>
<CircleButton 
    ...
    aria-label="1도 상태"
>
```

#### 3. `Lock` 컴포넌트
- 이 섹션이 "잠금 상태 정보"를 다루고 있음을 알려줍니다.
- `role`속성으로 중요한 상태 영역으로 인식하게 합니다.
```
<Content
    aria-label="잠금 상태"
>
    <Title>🔒잠금여부</Title>
    <LockBox
        $locked={true}
        role="status"
        aria-live="polite"
    >
        현재 기기가 잠금 상태입니다
    </LockBox>
</Context>
```
